## 🧠 System Design Snapshot

### 🏢 Enterprise .NET API Architecture (Advanced)

```id="archv2"
        Client (Angular / Blazor)
                  │
                  ▼
        API Gateway / Controllers
                  │
        ▼ Middleware Pipeline (.NET Hooks)
   ┌───────────────────────────────────────┐
   │ Logging │ Exception Handling │ Auth   │
   │ Rate Limiting │ CORS │ Validation     │
   └───────────────────────────────────────┘
                  │
        ┌─────────┴─────────┐
        ▼                   ▼
 Authentication        Business Logic Layer
 (JWT / Identity)      (Services / Rule Engine)
        │                   │
        ▼                   ▼
 Authorization        Caching Layer (Redis/In-Memory)
 (Role-based)               │
        │                   ▼
        │           Background Jobs (Hangfire)
        │                   │
        └─────────┬─────────┘
                  ▼
           Data Access Layer
     (Entity Framework / LINQ / Repositories)
                  │
        ┌─────────┴─────────┐
        ▼                   ▼
   MS SQL Server        External APIs
 (SPs, Indexing)       (3rd Party Services)
```

---

### ⚙️ Advanced Architecture Highlights

* Middleware Pipeline (Custom .NET Hooks for logging, validation, exception handling)
* Clean Architecture with Repository & Service Layers
* API Versioning & Contract-based Design
* Caching Strategy (In-Memory / Distributed)
* Background Processing with Hangfire Workers
* Resilience Patterns (Retry, Timeout handling for external APIs)
* Centralized Logging & Monitoring (structured logging approach)
* Secure Authentication & Authorization (JWT, Role-based access)
